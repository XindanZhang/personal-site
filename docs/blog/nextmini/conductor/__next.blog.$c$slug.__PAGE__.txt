<p>As the main entry of the dataplane, conductor is more like an orchestrator of each dataplane node.</p>
<h1 id="components-initialization">Components initialization</h1>
<p>Conductor initializes other dataplane components like <code>controller_interface</code>, <code>lossless_runtime</code>, <code>controller_reporter</code> and <code>flowstatsreporter</code>.</p>
<ul>
<li><code>controller_interface</code></li>
<li><code>lossless_runtime</code></li>
<li><code>controller_reporter</code></li>
<li><code>flowstatsreporter</code></li>
</ul>
<pre><code>  - ControllerReporterHandle in reporter.rs:25 is the low-level byte-metrics reporter.
    It is used by network interfaces in interface.rs:102: after packets are sent on a link, it aggregates bytes per flow_id and periodically sends
    DataplaneToController::Metrics in reporter.rs:112.
    Think: “how many bytes did this node send for each flow over each neighbor link?”
  - FlowStatsReporterHandle in flowstats.rs:77 is the higher-level flow lifecycle reporter.
    It is used by the local reader, routing table, user-space client, and lossless-unicast flow manager to report events like:
      - app flow start
      - user flow start
      - route assignment
      - flow finished
</code></pre>
<p><code>controller_interface</code> and <code>lossless_runtime</code> will be introduced later as these are two important components in DP.</p>
